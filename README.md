# SEKSOV Telegram Bot

Telegram-бот для личной фиксации введения одного препарата у нескольких пользователей. У каждого Telegram-пользователя отдельные партии раствора, история введений и остатки.

## Возможности

- создание новой партии раствора только если нет активной партии;
- фиксация введения кнопками `1 мл`, `2 мл`, `3 мл`;
- выбор только разрешённых способов и мест введения;
- просмотр текущего статуса партии;
- просмотр последнего введения с остатком после введения;
- просмотр истории с остатком после каждого введения;
- экспорт истории введений в CSV;
- отмена последней ошибочной записи с восстановлением остатка;
- отправка резервной копии SQLite-базы владельцу прямо из Telegram;
- опциональное красивое Telegram Mini App-приложение для визуального статуса, истории и партий;
- автоматический расчёт остатка текущей партии.

## Быстрый старт

### Windows / PowerShell

Самый простой способ — запускать готовый файл из корня проекта:

```powershell
.\start.ps1
```

Если PowerShell запрещает запуск скриптов, используйте:

```powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
```

Также можно запустить через cmd/bat:

```powershell
.\start.bat
```

Скрипт сам создаст `.venv`, установит зависимости из `requirements.txt`, создаст `.env` из `.env.example` при первом запуске и попросит заполнить `BOT_TOKEN` и настройки доступа (`AUTHORIZED_TELEGRAM_USER_IDS` или `REGISTRATION_CODE`, если доступ должен быть ограничен).

Важно: команды `source .venv/bin/activate` и `.venv/bin/activate` — это Linux/macOS, в PowerShell они не работают. Для ручного запуска на Windows используйте `.\.venv\Scripts\Activate.ps1`.

### Linux / macOS

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заполните BOT_TOKEN и при необходимости настройки доступа в .env
python run.py
```

Если проект установлен как пакет через `pip install -e .`, можно запускать и командой `seksov-bot`.

## Настройки

| Переменная | Описание | По умолчанию |
| --- | --- | --- |
| `BOT_TOKEN` | токен Telegram-бота от BotFather | обязательно |
| `DATABASE_PATH` | путь к SQLite-базе | `data/seksov.sqlite3` |
| `STANDARD_DOSE_ML` | стандартный объём одного введения | `1.0` |
| `AUTHORIZED_TELEGRAM_USER_IDS` | необязательный allowlist Telegram ID через запятую для пользователей, которым доступ открыт сразу | пусто |
| `ADMIN_TELEGRAM_USER_IDS` | Telegram ID владельцев, которым разрешён полный бэкап базы | пусто |
| `REGISTRATION_CODE` | необязательный код доступа: пользователь отправляет его боту один раз и сохраняется как разрешённый | пусто |
| `WEB_APP_URL` | HTTPS-ссылка на Telegram Mini App; если заполнена, бот покажет кнопку `✨ Красивое приложение` | пусто |
| `WEB_HOST` | адрес локального web-сервера Mini App | `127.0.0.1` |
| `WEB_PORT` | порт локального web-сервера Mini App | `8000` |
| `WEB_DEV_MODE` | режим локальной проверки Mini App через `?telegram_user_id=...`, не включать публично | `0` |


## Telegram Mini App / красивый интерфейс

В проект добавлен первый слой красивого приложения внутри Telegram: премиальный dashboard в стиле современных medical UI с карточкой текущей партии, крупным остатком, визуальным прогрессом, timeline-историей и списком партий. Основная фиксация введения остаётся в боте, потому что кнопки в чате быстрее и надёжнее, а Mini App используется для красивого просмотра и анализа.

Локальная проверка:

```bash
python run_web.py
```

На Windows можно запустить готовыми файлами:

```powershell
.\start_web.ps1
# или
.\start_web.bat
```

Для проверки без Telegram можно временно поставить `WEB_DEV_MODE=1` и открыть:

```text
http://127.0.0.1:8000/?telegram_user_id=ВАШ_TELEGRAM_ID
```

Для запуска именно внутри Telegram нужен публичный HTTPS-адрес web-сервера. Укажите его в `.env` как `WEB_APP_URL=https://...`, перезапустите бота, и в главной клавиатуре появится кнопка `✨ Красивое приложение`. Подпись Telegram WebApp проверяется на backend по `BOT_TOKEN`, поэтому данные пользователя не выдаются по неподписанному запросу; `WEB_DEV_MODE` используйте только локально.


## Разработка и проверки

Для запуска тестов установите dev-зависимости:

```bash
pip install -r requirements-dev.txt
pytest -q
```

Перед изменениями, связанными с web/Mini App, дополнительно проверяйте; `requirements-dev.txt` включает `httpx2`, который нужен текущему `starlette.testclient`:

```bash
python -m compileall -q src run.py run_web.py
git diff --check
```

## UX

Основные действия доступны с главной клавиатуры:

- `➕ Новая партия`
- `💉 1 мл`, `💉 2 мл`, `💉 3 мл` для быстрого выбора объёма введения
- `📊 Статус`
- `🏁 Завершить партию`
- `🕘 Последний приём`
- `📜 История`
- `📤 Экспорт CSV`
- `↩️ Отменить последнее`
- `💾 Бэкап`

Для безопасности личных медицинских записей можно не вносить каждого пользователя в `.env`: задайте `REGISTRATION_CODE`, отправьте код нужным людям, и каждый пользователь подключится сам, отправив код боту одним сообщением. Код перечитывается из `.env` при проверке доступа, поэтому если вы добавили код во время работы бота, пользователю достаточно отправить код ещё раз. Команда `/id` доступна даже без авторизации и показывает Telegram ID пользователя для ручного добавления. `AUTHORIZED_TELEGRAM_USER_IDS` остаётся для тех, кому доступ нужно выдать сразу.

После фиксации бот коротко показывает текущий приём, предыдущий приём, способ, место и остаток в текущей партии. История выводится компактным нумерованным списком с датой, объёмом, способом/местом и остатком после каждого введения; при необходимости её можно выгрузить кнопкой `📤 Экспорт CSV`. Если остатка активной партии недостаточно для выбранной кнопки, бот оставляет партию активной и предлагает выбрать меньший объём или завершить партию вручную, чтобы случайный тап не списывал пригодный остаток. При создании партии единицы препарата выбираются кнопками `мг`, `мкг`, `ЕД`, `мл`; при необходимости можно ввести свою единицу вручную. Если пользователь ошибся, кнопка `↩️ Отменить последнее` помечает последнюю запись отменённой и возвращает объём в партию; отменённые записи не показываются в обычной истории. Кнопка `💾 Бэкап` отправляет консистентный SQLite snapshot как резервную копию и доступна только владельцам из `ADMIN_TELEGRAM_USER_IDS`. Активную партию также можно завершить вручную кнопкой `🏁 Завершить партию`; история введений при этом сохраняется.
